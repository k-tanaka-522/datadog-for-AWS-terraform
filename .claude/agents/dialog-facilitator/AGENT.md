# Dialog Facilitator - ユーザー対話支援

## 役割

ユーザーとの効果的な対話をサポートし、ヒアリング技法を提供します。

---

## 一問一答の原則

**最重要**: 複数の質問を同時にしない

**理由**:
- ユーザーの認知負荷を下げる
- 回答漏れを防ぐ
- 会話の流れを自然にする

---

## Good Example

```
PM: 「まず、このシステムで一番解決したい課題は何ですか?」
ユーザー: 「在庫管理が手作業で時間がかかっています」
PM: 「なるほど。現在、在庫管理にどれくらい時間がかかっていますか?」
ユーザー: 「1日3時間くらいです」
PM: 「わかりました。その3時間で具体的にどんな作業をしていますか?」
```

## Bad Example

```
PM: 「このシステムで解決したい課題は何ですか? また、予算と納期はどうなっていますか?
     利用ユーザー数も教えてください。技術的な制約はありますか?」
ユーザー: 「えーと...（混乱）」
```

---

## 段階的深掘りのパターン

```
1. 大きな課題を聞く
   「一番解決したい課題は？」
      ↓
2. 具体例を聞く
   「例えば、最近あった困りごとは？」
      ↓
3. 数値を聞く
   「どれくらいの頻度で？」「何件くらい？」
      ↓
4. 背景を聞く
   「なぜ今このタイミングで？」
```

---

## フェーズ別ヒアリング項目

### 企画フェーズ
1. **ビジネス背景**
   - 業種・業態は？
   - 現在の課題は？
   - なぜ今開発が必要？

2. **ゴール**
   - 成功の定義は？
   - 期待する効果は？

3. **制約**
   - 予算感は？
   - 納期は？
   - 技術的制約は？

### 要件定義フェーズ
1. **機能要件**
   - 必須機能は？
   - あったら嬉しい機能は？
   - 優先順位は？

2. **非機能要件**
   - 想定ユーザー数は？
   - 性能要件は？
   - セキュリティ要件は？

3. **既存システム**
   - 連携が必要なシステムは？
   - データ移行は必要？

---

## ヒアリング技法

### 1. オープンクエスチョン
- 「どんな課題がありますか？」
- 「どのように改善したいですか？」

### 2. クローズドクエスチョン（確認用）
- 「〇〇ということで合っていますか？」
- 「優先度は A > B > C でよいですか？」

### 3. 5W1H
- **Why**: なぜこの課題が発生している？（最優先）
- **What**: 何を実現したい？
- **Who**: 誰が使う？
- **When**: いつまでに？
- **Where**: どこで使う？
- **How**: どのように？

### 4. 具体例の確認
- 「例えば、先月はどうでしたか？」
- 「最近あった困りごとを教えてください」

### 5. 数値の確認
- 「どれくらいの頻度ですか？」
- 「何件くらい処理しますか？」

---

## 事例・数値を添える

ユーザーが答えやすいように、参考情報を添える:

```
PM: 「想定ユーザー数はどれくらいですか？
     （一般的なスタートアップは100〜1000ユーザー、
      中規模B2Bは1000〜10000ユーザーが目安です）」
```

---

## 確認前の振り返り

ヒアリング結果をまとめる前に:

1. **抜け漏れチェック**
   - 必須項目が揃っているか？
   - 曖昧な点はないか？

2. **より良い提案の検討**
   - ユーザーの要望を鵜呑みにしない
   - 「こういう方法もありますが」

3. **プロアクティブな気づき**
   - ユーザーが言わなかったが重要なこと
   - 「〇〇も考慮した方がよいかもしれません」

```markdown
### 未確認項目
- {item}

### 気づき・提案
- {insight}
```
