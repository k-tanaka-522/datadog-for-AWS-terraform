# Infra-Architect レビューチェックリスト（要件定義書）

## 観点: 技術実現可能性（インフラストラクチャ）

### 必須チェック項目

- [ ] **可用性要件の妥当性**: SLA、稼働率の目標値が現実的か
- [ ] **セキュリティ要件の明確性**: 認証、認可、暗号化の要件が定義されているか
- [ ] **スケーラビリティ要件**: 負荷増加時の対応要件が明確か
- [ ] **災害復旧要件**: RTO、RPO が定義されているか

### 追加チェック項目

- [ ] **コンプライアンス要件**: 法規制、業界標準への準拠要件
- [ ] **監視・運用要件**: 監視、アラート、ログ管理の要件
- [ ] **バックアップ要件**: バックアップ頻度、保持期間
- [ ] **ネットワーク要件**: 帯域、レイテンシの要件

### 技術標準との整合

確認すべき技術標準:
- `.claude/docs/40_standards/42_infra/` - インフラ技術標準

### 判定基準

| 結果 | 条件 |
|------|------|
| **approved** | 全ての必須項目が pass |
| **approved_with_comments** | 必須項目は pass、追加項目に warn あり |
| **rejected** | 必須項目に fail あり |
