# Consultant レビューチェックリスト（要件定義書）

## 観点: ビジネス整合性

### 必須チェック項目

- [ ] **ビジネス要件との整合性**: ビジネス目標を達成できる要件か
- [ ] **機能要件の明確性**: 曖昧な点がないか、具体的に記述されているか
- [ ] **非機能要件の具体性**: 数値目標が明確か（性能、可用性、セキュリティ）
- [ ] **ステークホルダーの考慮**: 関係者の要望が反映されているか
- [ ] **リスク・制約の考慮**: 現実的な要件か、制約条件が明記されているか

### 追加チェック項目

- [ ] **優先順位**: 機能の優先順位が明確か
- [ ] **スコープ**: スコープ外が明確に定義されているか
- [ ] **用語の一貫性**: 用語が統一されているか
- [ ] **ユーザーストーリー**: エンドユーザー視点で要件が記述されているか

### 判定基準

| 結果 | 条件 |
|------|------|
| **approved** | 全ての必須項目が pass |
| **approved_with_comments** | 必須項目は pass、追加項目に warn あり |
| **rejected** | 必須項目に fail あり |
