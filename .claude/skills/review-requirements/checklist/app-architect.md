# App-Architect レビューチェックリスト（要件定義書）

## 観点: 技術実現可能性（アプリケーション）

### 必須チェック項目

- [ ] **機能要件の実現可能性**: 現在の技術で実現可能か
- [ ] **性能要件の妥当性**: 応答時間、スループットの目標値が現実的か
- [ ] **拡張性要件の妥当性**: 将来の拡張に対応できる設計が可能か
- [ ] **インテグレーション要件**: 外部システム連携の要件が明確か

### 追加チェック項目

- [ ] **技術的リスク**: 技術的なリスクが考慮されているか
- [ ] **既存システムとの整合**: 既存システムとの連携に問題はないか
- [ ] **データ要件**: データ量、データ形式の要件が明確か
- [ ] **API要件**: 必要なAPIが定義されているか

### 技術標準との整合

確認すべき技術標準:
- `.claude/docs/40_standards/41_app/` - アプリケーション技術標準

### 判定基準

| 結果 | 条件 |
|------|------|
| **approved** | 全ての必須項目が pass |
| **approved_with_comments** | 必須項目は pass、追加項目に warn あり |
| **rejected** | 必須項目に fail あり |
