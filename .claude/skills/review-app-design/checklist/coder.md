# Coder レビューチェックリスト（アプリ設計書）

## 観点: 実装可能性

### 必須チェック項目

- [ ] **実装可能性**: この設計で実装できるか
- [ ] **技術的課題**: 実装時に困りそうな点はないか
- [ ] **API設計の妥当性**: エンドポイント、リクエスト/レスポンス形式が明確か
- [ ] **データモデルの妥当性**: ER図、リレーション設計が適切か
- [ ] **技術標準との整合**: `.claude/docs/40_standards/41_app/` に準拠しているか

### 追加チェック項目

- [ ] **非機能要件の考慮**: 性能、スケーラビリティへの配慮があるか
- [ ] **エラーハンドリング方針**: 異常系の処理方針が定義されているか
- [ ] **依存関係**: 外部ライブラリ、サービスへの依存が明確か
- [ ] **テスト容易性**: テストしやすい設計になっているか

### 技術標準参照

確認すべき技術標準:
- `.claude/docs/40_standards/41_app/` - アプリケーション技術標準
- 言語別標準（TypeScript, Python, Go など）

### 判定基準

| 結果 | 条件 |
|------|------|
| **approved** | 全ての必須項目が pass |
| **approved_with_comments** | 必須項目は pass、追加項目に warn あり |
| **rejected** | 必須項目に fail あり |
