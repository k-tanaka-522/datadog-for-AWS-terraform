# Consultant レビューチェックリスト（アプリ設計書）

## 観点: ビジネス要件整合

### 必須チェック項目

- [ ] **ビジネス要件との整合**: 要件定義の内容が設計に反映されているか
- [ ] **ユーザー体験**: エンドユーザーにとって使いやすい設計か
- [ ] **機能網羅性**: 要件定義の機能が全て設計に含まれているか

### 追加チェック項目

- [ ] **コスト妥当性**: ビジネス価値に見合った設計か
- [ ] **優先順位の反映**: 要件の優先順位が設計に反映されているか
- [ ] **将来の拡張性**: ビジネスの成長に対応できる設計か

### 参照ドキュメント

- `docs/requirements/` - 要件定義書
- ビジネス要件との整合性を確認

### 判定基準

| 結果 | 条件 |
|------|------|
| **approved** | 全ての必須項目が pass |
| **approved_with_comments** | 必須項目は pass、追加項目に warn あり |
| **rejected** | 必須項目に fail あり |
