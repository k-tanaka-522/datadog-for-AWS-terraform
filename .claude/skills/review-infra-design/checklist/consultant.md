# Consultant レビューチェックリスト（インフラ設計書）

## 観点: ビジネス要件整合

### 必須チェック項目

- [ ] **ビジネス要件との整合**: 要件定義の非機能要件が設計に反映されているか
- [ ] **SLA達成可能性**: 要件のSLAを達成できる設計か
- [ ] **コスト妥当性**: ビジネス価値に見合ったインフラ投資か

### 追加チェック項目

- [ ] **将来の拡張性**: ビジネスの成長に対応できる設計か
- [ ] **運用コスト**: 長期的な運用コストが考慮されているか
- [ ] **リスク対応**: ビジネスリスクへの対応策があるか

### 参照ドキュメント

- `docs/requirements/` - 要件定義書（非機能要件）
- ビジネス要件との整合性を確認

### 判定基準

| 結果 | 条件 |
|------|------|
| **approved** | 全ての必須項目が pass |
| **approved_with_comments** | 必須項目は pass、追加項目に warn あり |
| **rejected** | 必須項目に fail あり |
