# QA レビューチェックリスト（コード）

## 観点: テスト可能性・品質

### 必須チェック項目

- [ ] **テスト可能性**: このコードはテストしやすいか
- [ ] **品質**: コードの可読性、保守性が確保されているか
- [ ] **エラーハンドリング**: 例外処理が適切か
- [ ] **設計書との整合性**: 基本設計書の通りに実装されているか
- [ ] **技術標準準拠**: コーディング規約に準拠しているか

### セキュリティチェック（OWASP Top 10）

- [ ] **SQLインジェクション対策**: パラメータ化クエリを使用しているか
- [ ] **XSS対策**: 出力のエスケープが適切か
- [ ] **認証・認可**: 認証・認可の実装が適切か
- [ ] **機密情報**: 機密情報がハードコードされていないか
- [ ] **入力検証**: ユーザー入力の検証が適切か

### 追加チェック項目

- [ ] **関数の長さ**: 関数が長すぎないか（推奨: 50行以内）
- [ ] **ネストの深さ**: ネストが深すぎないか（推奨: 3階層以内）
- [ ] **単一責任**: クラス/関数が単一責任の原則に従っているか
- [ ] **命名**: 変数、関数名が分かりやすいか
- [ ] **コメント**: 必要なコメントが記述されているか

### 技術標準参照

確認すべき技術標準:
- `.claude/docs/40_standards/41_app/` - アプリケーション技術標準
- 言語別標準（TypeScript, Python, Go など）

### 判定基準

| 結果 | 条件 |
|------|------|
| **approved** | 全ての必須項目が pass |
| **approved_with_comments** | 必須項目は pass、追加項目に warn あり |
| **rejected** | 必須項目に fail、またはセキュリティ項目に fail あり |
