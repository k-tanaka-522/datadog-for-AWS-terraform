# Coder レビューチェックリスト（テストコード）

## 観点: カバレッジ・実装整合

### 必須チェック項目

- [ ] **カバレッジ**: 実装コードを十分カバーしているか（推奨: 80%以上）
- [ ] **実装との整合性**: API設計、データモデルと一致しているか
- [ ] **境界値テスト**: 境界条件がテストされているか
- [ ] **異常系テスト**: エラーケースがテストされているか
- [ ] **テストの独立性**: 他のテストに依存していないか

### 追加チェック項目

- [ ] **テストの可読性**: テストコードが理解しやすいか
- [ ] **テストデータ**: テストデータが適切に管理されているか
- [ ] **モック/スタブ**: 外部依存が適切にモックされているか
- [ ] **テスト命名**: テスト名が何をテストしているか明確か
- [ ] **アサーション**: アサーションが適切で十分か

### テスト種別チェック

- [ ] **ユニットテスト**: 単体テストが実装されているか
- [ ] **インテグレーションテスト**: 統合テストが実装されているか
- [ ] **E2Eテスト**: 必要に応じてE2Eテストが実装されているか

### 技術標準参照

確認すべき技術標準:
- `.claude/docs/40_standards/41_app/` - アプリケーション技術標準
- テストフレームワーク固有の規約

### 判定基準

| 結果 | 条件 |
|------|------|
| **approved** | 全ての必須項目が pass |
| **approved_with_comments** | 必須項目は pass、追加項目に warn あり |
| **rejected** | 必須項目に fail あり |
